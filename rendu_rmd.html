Dfinition du modles

Formalisme mathmatique

Modle de comptition Lotka-Volterra :

\[\dfrac{dn_i}{dt}=rn_i(1-\dfrac{\sum_{j=1}^Na(x_i,x_j)n_j}{K(x_i)}) \] \[i = 1,2,...,N\]

Dfinition des paramtres

Avec :

  • \(i\) le nombre de phnotype

  • \(r\) le taux de dveloppement de ce phnotype

  • \(a(x_i,x_j)\) la fonction de comptition interspcifique entre un phnotype \(x_i\) et un phnotype \(x_j\). Lintensit de la comptition ne dpend que de la distance entre les deux phnotype d ($ x_i-x_j $). Si d augmente la comptition augmente. Donc plus les traits sont proche plus la comptition est importante

  • \(K(x_i)\) la fitness dun individu qui porte le phnotype \(x_i\)

Implmentation du modle sous R

Sous R, pour des raisons doptimisation, la proportion de chaque phnotypes sera calcul avec la formule suivente : \[\dfrac{dN}{dt} = r*N_0*(1-N_0*M)\] Avec :

  • \(N_0\) : les valeurs initiales de proportions des phnotypes

  • \(M\) : la matrice qui regroupe les valeurs des fonctions de comptitions \(a\) entre chaque phnotypes et de fitness \(K\) gal : \[ M_{ij} = \begin{pmatrix} \dfrac{a_{11}}{K_1} & \cdots & \dfrac{a_{N1}}{K_1}\\ \vdots & \space & \vdots \\ \dfrac{a_{1N}}{K_N} & \cdots & \dfrac{a_{NN}}{K_N} \end{pmatrix} \]

Cration des fonctions ncessaires la simulation

fonc.K = function(x, K0 = K0, lambda = lambda, x0 = x0){return(
  max(0, 1/(K0-lambda*(x - x0)^2)))}  #Afin de ne pas obtenir de valeur ngative, le maximum entre 0 et la valeur de K est retourn. 

fonc.a = function(x, y, sigma = sigma){
  return(exp(-(x-y)**2/2*(sigma^2)))
}

EDO_LK <- function(t, y, paraM){
  with(as.list(paraM), { # permet d'viter de lister les paramtres 

  Mat = A * K
  
  dndt <- r*y*(1-(y%*%Mat)) # Equation diffrentielle 
  
  return(list(dndt))
  })
}

Introduction de Mutations et fixation des traits

Une mutation est introduite dans le modle. Une fraction de la population va tre soumise cette lgre modification de la valeur du trait, avec une probabilit \(p\). Nous avons choisi de ne faire que des mutations droite, cest dire quil ny a pas de retour en arrire possible, la valeur du trait mute toujours en augmentant. Il sagit dune hypothse forte sans relle valeur biologique, en effet, lvolution peut tout fait faire disparatre une mutation apparue pralablement. Nous nous sommes arrtes sur cette hypothse par mesure de simplicit.

Pour tous les traits deffectifs initiaux quivalent :

En concervant lhypothse qu lorigine, la population comporte autant dindividu que de phnotypes possible, voici-ce que lon observe.

#Initialisation des paramtres pour cette tude
sigma = 0.9
N = 20 #nombre de traits
M = 100 #temps volutif rptition du temps t.

temps <- seq(0,100,1)

#Cration des matrice K et A (elles restent fixe malgr la mutation)
K = matrix(NA, nrow = length(X), ncol = length(X))
A = matrix(NA, nrow = length(X), ncol = length(X))

for(i in 1:length(X)){
  for(j in 1:length(X)){
    K[i,j] =  fonc.K(x = X[j],K0 = K0, lambda = lambda, x0 = x0)
    A[i,j] = fonc.a(x = X[i], y = X[j], sigma = sigma)
  }
}

paraM = list(r = r, K=K, A=A)

# Cration de la matrice qui va contenir les valeurs des phnotypes au cours du temps volutif M
nTmat = matrix(nrow=M, ncol=length(X))

for(m in 1:M){
  solution <- ode(y = nbarre, times = temps, func = EDO_LK, parms = paraM)
  nT = solution[nrow(solution),2:ncol(solution)]
  
  for (rgtrait in 1:length(nT)){
    if (nT[rgtrait] > 0.01){
      nTmute <- nT[1:length(nT)-1]*p
      a = runif(1,0,1)
      
      if (a>0.5){ #mutation  droite
        nT[1:length(nT)-1] = nT[1:length(nT)-1] - nTmute
        nT[2:length(nT)] = nT[2:length(nT)] + nTmute
      }
      #else{ #mutation  gauche
        #nT[1:length(nT)-1] = nT[1:length(nT)-1] + nTmute
        #nT[2:length(nT)] = nT[2:length(nT)] - nTmute
      #}
    }
  }
  nbarre <- nT
  nTmat[m,] <- nT
}
